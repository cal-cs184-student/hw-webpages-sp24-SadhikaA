Task 1: Drawing Single-Color Triangles

Rasterizing Triangles

In order to rasterize triangles to the frame buffer, we check if the center of each pixel is within the triangle. The rasterize_triangle function takes in coordinates for each of the three corners of the triangle. My implementation started off by traversing through the frame buffer and then determining whether the point was inside the line by creating a line function for each of the two corners of the triangle. After making sure that the center of the point was above each of the lines, it would fill the pixel in. This handled coordinates received in the clockwise direction, but to include coordinates in the counter-clockwise direction, I checked to see if the center of the point was under all three lines. In order to speed up computation, I used a bounding box around the triangle which was calculated by determining the lowest and highest x and y values that the triangle covered.

This algorithm is no worse than traversing each sample individually in the bounding box of the triangle and determining if it's in the triangle because we still iterate through all the points in the bounding box. There is no optimization or short circuiting that reduces the number of pixels traversed.
